// Performance Benchmarking Samples

These examples are built to support basic performance benchmarking for application monitoring capabilities.

The following parameters are supported, and will influence the benchmark executed:

  • iterations - Controls the number of iteration we'll execute when running the benchmark
  • complexity - Controls the number of recursive function calls we'll execute, to grow the complexity of the call stack
  • rate - Controls the variance of the default response-time within each benchmarking loop. Valid values are .1 (10% of default) to 5 (500% of default)
